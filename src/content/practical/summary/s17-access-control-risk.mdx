> 섹션 17은 접근통제(DAC·MAC·RBAC) + 인증(다요소인증·생체인증) + 위험관리(ALE·SLE·ARO) + BCP/DRP(RTO·RPO) + 포렌식 + CC(EAL) + ISMS-P가 핵심이며, 시험에서 서술형 고배점으로 가장 많이 출제되는 섹션입니다. 특히 DAC vs MAC vs RBAC 비교, SLE·ALE 계산 문제, RTO·RPO·DRS 4종류 비교는 매 회차 거의 빠짐없이 출제됩니다.

## 1. 개념 및 배경

정보보안 일반/관리는 기술적 보안(암호·인증·접근통제)과 관리적 보안(위험관리·BCP·법제도)을 아우르는 과목으로, 실무와 시험 모두 중요한 기반 지식입니다. 단순 암기보다 개념 간 인과관계(왜 MAC이 군사 환경에 적합한가, 왜 BCP가 먼저 수행되는가 등)를 이해하는 것이 고득점의 핵심입니다.

## 2. 동작 원리 / 구조

### 🔐 1. 접근통제 모델 ⭐⭐⭐

**3대 접근통제 정책 비교**

| 구분 | DAC | MAC | RBAC |
| --- | --- | --- | --- |
| 풀이 | Discretionary Access Control | Mandatory Access Control | Role-Based Access Control |
| 주체 | 자원 소유자(Owner)가 접근 권한 결정 | 관리자(시스템)가 보안 레이블로 결정 | 역할(Role)에 따라 권한 할당 |
| 메커니즘 | ACL(접근제어 목록), ID 기반 | 보안 레이블(등급), 격자 기반 | 역할 → 권한 → 사용자 매핑 |
| 장점 | 유연성·편의성 | 엄격한 보안 (정보 유출 방지) | 관리 편의성, 최소권한 원칙 |
| 단점 | Trojan Horse 공격 취약 | 구현 복잡, 경직성 | 역할 폭발(Role Explosion) |
| 적용 | 일반 상업 환경 | 군사·정부 기밀 환경 | 기업 환경 (가장 널리 사용) |
| 보안 모델 | - | BLP, Biba, Clark-Wilson, Chinese Wall | - |

**추가 접근통제 유형**

- **ABAC (Attribute-Based AC)**: 속성(시간, 위치, 부서 등) 기반 — 가장 유연, 최신 트렌드
- **ACM (Access Control Matrix)**: 행=주체, 열=객체의 권한 행렬

### 🏰 2. MAC 보안 모델 4종 ⭐⭐⭐

**핵심 보안 모델 비교**

| 모델 | 연도 | 핵심 원칙 | 특징 |
| --- | --- | --- | --- |
| BLP (Bell-LaPadula) | 1970 | No Read Up / No Write Down | 기밀성 보호, 군사 모델 |
| Biba | - | No Read Down / No Write Up | 무결성 보호, BLP와 반대 |
| Clark-Wilson | - | 무결성 3요소 (인증·검증·제어) | 상업 환경의 무결성 모델 |
| Chinese Wall | - | 이해충돌 방지 | 금융·컨설팅 환경 (과거 접근 이력 기반) |

**BLP vs Biba 직관적 이해**

```text
BLP (기밀성 보호):
- No Read Up → 낮은 등급이 높은 등급 문서 읽기 금지
- No Write Down → 높은 등급이 낮은 등급에 쓰기 금지
  (기밀 내용이 낮은 곳으로 흘러내려가면 안 됨)

Biba (무결성 보호):
- No Read Down → 높은 등급이 낮은 등급 문서 읽기 금지
- No Write Up → 낮은 등급이 높은 등급에 쓰기 금지
  (신뢰도 낮은 데이터가 위로 올라가면 안 됨)
```

### 👤 3. 인증 (Authentication) ⭐⭐⭐

**인증 5가지 유형 (Type 암기!)**

| Type | 구분 | 예시 |
| --- | --- | --- |
| Type 1 | 알고 있는 것 (지식) | 패스워드, PIN |
| Type 2 | 가지고 있는 것 (소유) | OTP, 스마트카드, 토큰 |
| Type 3 | 본인 자체 (존재/생체) | 지문, 홍채, 얼굴, 정맥 |
| Type 4 | 위치 기반 | IP, MAC, Callback |
| Type 5 | 행동 기반 | 서명, 키보드 패턴 |

**다요소 인증 (MFA)**

```text
2-Factor 인증: 서로 다른 Type 2가지 이상 조합
      예: 패스워드(Type 1) + OTP(Type 2) = 2FA ✅
      예: 패스워드(Type 1) + PIN(Type 1) = 2FA ❌ (같은 Type)
```

**생체인증 오류율**

| 지표 | 설명 | 관계 |
| --- | --- | --- |
| FAR (False Acceptance Rate) | 타인을 본인으로 잘못 수락 | 보안에 치명적 |
| FRR (False Rejection Rate) | 본인을 타인으로 잘못 거부 | 편의성 저하 |
| CER/EER | FAR = FRR이 되는 교차점 | 낮을수록 정확한 시스템 |

> FAR ↓ 설정 → FRR ↑ (보안 강화, 편의성 저하), FAR ↑ 설정 → FRR ↓ (편의성 강화, 보안 취약)

### 📊 4. 위험관리 (Risk Management) ⭐⭐⭐

**위험 관리 핵심 공식 (계산 문제 필수!)**

```text
SLE = AV × EF
ALE = SLE × ARO
대책 도입 타당성 = ALE 이전 − ALE 이후 − 대책 비용
```

| 용어 | 풀이 | 의미 |
| --- | --- | --- |
| AV | Asset Value | 자산 가치 |
| EF | Exposure Factor | 위협 발생 시 손실 비율 (0~1) |
| SLE | Single Loss Expectancy | 1회 발생 시 예상 손실액 |
| ARO | Annual Rate of Occurrence | 연간 발생 횟수 |
| ALE | Annual Loss Expectancy | 연간 예상 손실액 |

**위험 처리 4가지 전략**

| 전략 | 설명 | 예시 |
| --- | --- | --- |
| 위험 수용 (Risk Acceptance) | 위험을 감수하고 수용 | 비용 대비 손실이 적을 때 |
| 위험 감소 (Risk Reduction) | 보안 대책 적용으로 위험 줄임 | 방화벽, IDS 도입 |
| 위험 전가 (Risk Transfer) | 제3자에게 위험 이전 | 보험, 아웃소싱 |
| 위험 회피 (Risk Avoidance) | 위험 원인 자체를 제거 | 서비스 중단 |

**위험 평가 방법론**

- **정량적 방법**: ALE 등 수치(금액)로 표현 → 객관적이나 시간·비용 소요
- **정성적 방법**: H/M/L 등급으로 표현 → 신속하나 주관적
- **복합(혼합)**: 두 방법을 결합한 현실적 접근

> 🔍 **보완**: 세부 기법 (서술형 대비) — **정량적**: 과거자료 분석법, 수학공식 접근법, 확률분포 추정법, 몬테카를로 시뮬레이션. **정성적**: 델파이법(전문가 합의), 시나리오법, 순위결정법(Ranking), 브레인스토밍. 정량은 "숫자·금액·객관", 정성은 "전문가·시나리오·주관"으로 구분해 기억하세요.

### 🆘 5. BCP / DRP ⭐⭐⭐

**BCP vs DRP 개념**

| 구분 | BCP (Business Continuity Plan) | DRP (Disaster Recovery Plan) |
| --- | --- | --- |
| 목적 | 업무 연속성 유지 (광의의 개념) | IT 시스템 복구 (BCP의 일부) |
| 수립 절차 | BIA → 전략 수립 → 계획 → 교육·훈련 → 유지·개선 (5단계) | RTO·RPO 목표 기반 복구 전략 |

**BIA (Business Impact Analysis)**

- BCP 수립의 첫 번째 단계
- 업무 중단 시 영향도·복구 우선순위 분석
- RTO, RPO 목표값 도출

**핵심 지표**

| 지표 | 풀이 | 의미 |
| --- | --- | --- |
| RTO | Recovery Time Objective | 목표 복구 시간 (업무 중단 후 복구까지 허용 시간) |
| RPO | Recovery Point Objective | 목표 복구 시점 (어느 시점의 데이터까지 복구 허용) |
| MTD | Maximum Tolerable Downtime | 최대 허용 중단 시간 (이 시간을 넘기면 조직에 치명적) |
| WRT | Work Recovery Time | 복구 후 정상 업무 재개까지 걸리는 시간 |

> 🔍 **보완**: 시간 축 관계 — **MTD = RTO + WRT**. RTO는 시스템을 복구하는 시간, WRT는 복구된 시스템에서 데이터 검증·업무 재개까지의 시간이며, 둘의 합이 MTD를 넘으면 안 됩니다. RPO는 "얼마나 과거의 데이터까지 복구하느냐"(허용 데이터 손실량)로, 백업 주기가 짧을수록 RPO가 0에 가까워집니다.

**DRS (재해복구 사이트) 4종류 ⭐⭐⭐**

| 유형 | RTO | 특징 |
| --- | --- | --- |
| Mirror Site | 즉시 (RTO = 0) | 주 시스템과 동일한 IT 자원 운영, 실시간 데이터 동기화, 비용 최고 |
| Hot Site | 수 시간 내 | IT 자원 대기 상태로 준비, Standby/Mirroring |
| Warm Site | 수일 내 | IT 자원 일부 준비, 일정 작업 후 운영 가능 |
| Cold Site | 수주 내 | 건물·전력만 확보, IT 자원은 재해 후 도입, 비용 최저 |

> 📌 RTO 순서 암기: Mirror < Hot < Warm < Cold (복구 시간 오름차순)

### 🔍 6. 디지털 포렌식 ⭐⭐⭐

**포렌식 5대 원칙**

- **정당성**: 적법한 절차로 수집된 증거만 유효
- **재현성**: 동일 조건으로 동일 결과 재현 가능
- **신속성**: 휘발성 증거를 빠르게 수집
- **연계보관성 (Chain of Custody)**: 증거 수집~법정까지 취급 이력 기록
- **무결성**: 증거가 훼손·변조되지 않음 (해시값으로 검증)

**포렌식 증거 수집 순서**

```text
1. 휘발성 데이터 수집 (메모리, 네트워크 연결, 실행 프로세스)
2. 비휘발성 데이터 수집 (하드디스크 이미징)
3. 이미지 무결성 검증 (MD5/SHA 해시)
4. 분석 (사본으로만 분석, 원본 보존)
5. 보고서 작성
```

**Anti-Forensics (안티포렌식)**

- 증거 은폐·변조를 위해 디스크 덮어쓰기(0 또는 1로), 파일 삭제, 타임스탬프 변조
- 타임스탬프 변조 명령: `date -t [시간]`, `touch -t [시간]` 으로 MAC time 조작

**Windows 사고 조사 명령어**

```text
hostname / whoami / ver / systeminfo
ipconfig /all
netstat -anob
net use / net share / net session
tasklist /v
ipconfig /displaydns
arp -a
netstat -rn / route print
```

### 📜 7. CC (Common Criteria) ⭐⭐

**CC 개요**

| 항목 | 내용 |

| 정식 명칭 | Common Criteria (공통평가기준) |

**EAL 등급 의미**

- **EAL1**: 기능 시험 (가장 낮음)

- **EAL7**: 형식적 검증 (가장 높음, 군사급)

### 🏛️ 9. ISMS-P ⭐⭐⭐

**ISMS-P 핵심 구조**

| 항목 | 내용 |
| --- | --- |
| 인증 체계 | 정보보호 관리체계(ISMS) + 개인정보보호(P) 통합 |
| 관리 사이클 | PDCA (Plan → Do → Check → Act) |
| 인증 기준 | 3개 영역, ISMS-P 총 101개 통제항목 (관리체계 16 + 보호대책 64 + 개인정보 21) |
| 물리보안 분야 | 2.4 물리 보안 (2.4.1~2.4.7) |

> 🔍 **보완**: "64개"는 ②**보호대책 요구사항** 분야 항목 수일 뿐 전체가 아닙니다. 전체 통제항목은 **ISMS-P 101개**(2023.11 개정, 이전 102개), **ISMS 80개**입니다. "16개 분야"는 보호대책 요구사항의 세부 분야 수를 가리킵니다. 인증 유효기간은 3년, 연 1회 이상 사후심사입니다.

**2.4 물리 보안 7개 통제항목 암기**

```text
2.4.1 보호구역 지정
2.4.2 출입통제
2.4.3 Rack 및 단말기 보호
2.4.4 UPS, 항온항습 등 환경 보호
2.4.5 보호구역 내 작업
2.4.6 반출입 기기 통제
2.4.7 Clean Desk, 정보 출력·복사물 보호
```

## 3. 예시 / 적용 시나리오

### 🎯 기출: SLE·ALE 계산 문제

**실제 기출 유형**

```text
Q. 자산가치 100만원, 손실 비율 20%, 연간 발생 횟수 0.5회,
   보안 대책 비용 연 1.7만원일 때 도입 타당성을 계산하시오.

풀이:
1) SLE = AV × EF = 100만 × 0.2 = 20만원
2) ALE(이전) = SLE × ARO = 20만 × 0.5 = 10만원

대책 도입 후:
3) ALE(이후) = 3만원 (가정)
4) 타당성 = ALE이전 - ALE이후 - 대책비용
          = 10만 - 3만 - 1.7만 = 5.3만원 > 0 → 도입 타당!
```

### 🎯 기출: DAC·MAC·RBAC 비교 서술형

**실제 기출 유형**

```text
Q. DAC, MAC, RBAC의 차이점을 설명하시오.

DAC(임의적 접근통제):
- 자원 소유자가 접근 권한을 자율적으로 설정
- ACL 기반, 유연하나 Trojan Horse 취약

MAC(강제적 접근통제):
- 시스템 관리자가 보안 레이블로 강제 적용
- BLP(기밀성), Biba(무결성) 모델
- 군사·정부 환경에 적합

RBAC(역할기반 접근통제):
- 역할에 권한 부여 → 사용자를 역할에 할당
- 최소권한 원칙 준수, 기업 환경에 가장 널리 사용
- DAC의 단점 보완
```

### 🎯 기출: DRS 4종 비교 + RTO 서술형

**실제 기출 유형**

```text
Q. 재해복구 시스템 유형 4가지를 RTO 기준으로 비교하시오.

Mirror Site: RTO=0, 실시간 동기화, 비용 최고
Hot Site:    RTO=수 시간, 준비된 자원 Standby
Warm Site:   RTO=수 일, 일부 자원만 준비
Cold Site:   RTO=수 주, 공간·전력만 확보, 비용 최저

→ 비용과 복구시간은 반비례 관계
```

## 4. 실무 및 시험 포인트 정리

| 주제 | 핵심 암기 | 출제 빈도 |
| --- | --- | --- |
